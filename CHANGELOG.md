# Changelog — Goodveen 2.0

## 2026-05-11 — User Account & Auth Improvements

### ✅ Завершённые задачи

#### 1. **Восстановление пароля**
- ✅ Backend: `POST /api/auth/request-reset` — генерирует токен и логирует ссылку для сброса
- ✅ Backend: `POST /api/auth/reset-password` — проверяет токен и обновляет пароль
- ✅ Frontend: методы `authApi.requestPasswordReset()` и `authApi.resetPassword()`
- ✅ Страница `/reset-password` для завершения сброса пароля из email
- ✅ Интеграция в `AuthPopup` с реальным API вызовом
- ✅ Валидация токена (30 минут жизни, одноразовое использование)

**Как работает:**
1. Пользователь вводит email в форме "Забыли пароль?"
2. Сервер генерирует токен, хеширует его и сохраняет в БД
3. В консоли сервера логируется ссылка вида: `http://localhost:3000/reset-password?token=...`
4. Пользователь переходит по ссылке, вводит новый пароль
5. Токен проверяется и помечается как использованный

### 🧪 Как протестировать

#### Восстановление пароля:
1. Откройте `/password-reset` или нажмите "Забыли пароль?" в форме входа
2. Введите email зарегистрированного пользователя
3. Проверьте консоль сервера — там будет ссылка для сброса
4. Перейдите по ссылке, введите новый пароль
5. Попробуйте войти с новым паролем

#### Checkout авторизация:
1. Добавьте товар в корзину и перейдите в `/checkout`
2. Выберите "У меня есть аккаунт"
3. Введите email и пароль, нажмите "Войти"
4. Проверьте что форма автоматически заполнилась вашими данными
5. Нажмите "Кабинет" чтобы перейти в личный кабинет

#### Адреса доставки:
1. Войдите в аккаунт и перейдите в `/cabinet?tab=delivery`
2. Нажмите "Добавить новый"
3. Заполните форму и сохраните
4. Попробуйте отредактировать адрес
5. Создайте второй адрес и установите его по умолчанию
6. Проверьте что бейдж "По умолчанию" переместился
7. Удалите один из адресов
