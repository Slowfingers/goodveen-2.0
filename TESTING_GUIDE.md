# Руководство по тестированию — Goodveen 2.0

## 🚀 Быстрый старт

### 1. Запуск проекта

```bash
# Терминал 1 — Backend
cd server
npm run dev

# Терминал 2 — Frontend
npm run dev
```

Откройте http://localhost:5173

---

## 🧪 Сценарии тестирования

### Сценарий 1: Регистрация и вход

1. **Регистрация нового пользователя**
   - Нажмите "Войти" в хедере
   - Переключитесь на "Регистрация"
   - Заполните: email, пароль, имя
   - Нажмите "Зарегистрироваться"
   - ✅ Должны автоматически войти в систему

2. **Выход из системы**
   - Нажмите на иконку пользователя в хедере
   - Нажмите "Выйти"
   - ✅ Должны вернуться на главную страницу

3. **Вход в систему**
   - Нажмите "Войти" в хедере
   - Введите email и пароль
   - Нажмите "Войти"
   - ✅ Должны войти в систему

---

### Сценарий 2: Восстановление пароля

1. **Запрос на восстановление**
   - Нажмите "Войти" → "Забыли пароль?"
   - Введите email зарегистрированного пользователя
   - Нажмите "Отправить ссылку"
   - ✅ Должно показаться сообщение "Ссылка отправлена"

2. **Проверка консоли сервера**
   - Откройте терминал где запущен backend
   - Найдите строку: `Reset link: http://localhost:3000/reset-password?token=...`
   - Скопируйте ссылку

3. **Сброс пароля**
   - Вставьте ссылку в браузер (замените порт на 5173 если нужно)
   - Введите новый пароль дважды
   - Нажмите "Сохранить пароль"
   - ✅ Должно показаться "Пароль изменён"
   - ✅ Через 3 секунды перенаправит на главную

4. **Проверка нового пароля**
   - Попробуйте войти со старым паролем → должна быть ошибка
   - Войдите с новым паролем → должно работать

---

### Сценарий 3: Личный кабинет — Персональные данные

1. **Открытие кабинета**
   - Войдите в систему
   - Нажмите на иконку пользователя → "Личный кабинет"
   - ✅ Должна открыться вкладка "Личные данные"

2. **Редактирование профиля**
   - Измените имя на "Александр Тестовый"
   - Введите телефон "+998 90 123 45 67"
   - Нажмите "Сохранить"
   - ✅ Должно показаться "Сохранено"
   - ✅ Обновите страницу — данные должны сохраниться

3. **Смена пароля**
   - Введите текущий пароль
   - Введите новый пароль (минимум 6 символов)
   - Нажмите "Сменить пароль"
   - ✅ Должно показаться "Пароль изменён"
   - Выйдите и войдите с новым паролем

---

### Сценарий 4: Адреса доставки

1. **Создание первого адреса**
   - В личном кабинете перейдите на вкладку "Доставка"
   - Нажмите "Добавить новый"
   - Заполните форму:
     - Название: "Дом"
     - Улица: "Узбекистон Овози 2/1"
     - Город: "Tashkent"
     - Район: "Мирабад"
   - ✅ Чекбокс "По умолчанию" должен быть включён автоматически
   - Нажмите "Сохранить"
   - ✅ Адрес должен появиться в списке с бейджем "По умолчанию"

2. **Создание второго адреса**
   - Нажмите "Добавить новый"
   - Заполните форму:
     - Название: "Офис"
     - Улица: "Амир Темур 107"
     - Город: "Tashkent"
   - Включите чекбокс "Использовать по умолчанию"
   - Нажмите "Сохранить"
   - ✅ Бейдж "По умолчанию" должен переместиться на новый адрес

3. **Редактирование адреса**
   - Нажмите "Изменить" на любом адресе
   - Измените название на "Работа"
   - Нажмите "Сохранить"
   - ✅ Изменения должны сохраниться

4. **Удаление адреса**
   - Нажмите "Удалить" на любом адресе
   - Подтвердите удаление
   - ✅ Адрес должен исчезнуть из списка

---

### Сценарий 5: История заказов

1. **Просмотр заказов**
   - В личном кабинете перейдите на вкладку "Заказы"
   - ✅ Если заказов нет — должно показаться "У вас пока нет заказов"
   - ✅ Если есть заказы — должны отображаться с реальными данными

2. **Создание тестового заказа** (для проверки)
   - Добавьте товар в корзину
   - Перейдите в Checkout
   - Оформите заказ
   - Вернитесь в кабинет → вкладка "Заказы"
   - ✅ Новый заказ должен появиться в списке

---

### Сценарий 6: Checkout с авторизацией

1. **Checkout как гость**
   - Выйдите из системы
   - Добавьте товар в корзину
   - Перейдите в Checkout
   - Выберите "Новый покупатель"
   - ✅ Должны показаться поля: Имя, Email, Телефон

2. **Вход в Checkout**
   - Выберите "У меня есть аккаунт"
   - Введите email и пароль
   - Нажмите "Войти"
   - ✅ Форма должна автоматически заполниться вашими данными
   - ✅ Кнопка "Выйти" должна смениться на "Кабинет"

3. **Переход в кабинет из Checkout**
   - Нажмите "Кабинет"
   - ✅ Должен открыться личный кабинет

## 🐛 Известные ограничения

1. **Email не отправляется**
   - Ссылки для восстановления пароля логируются в консоль сервера
   - Для production нужно интегрировать email-сервис

2. **Банковские карты**
   - Не реализованы (не требовались в задаче)
   - Можно добавить позже по аналогии с адресами

3. **Выбор адреса в Checkout**
   - Пока нет выбора сохранённого адреса при оформлении
   - Можно добавить дропдаун с адресами пользователя

## 🔍 Отладка

### Проверка токена восстановления пароля
```sql
-- В Supabase SQL Editor
SELECT * FROM password_reset_tokens 
WHERE used_at IS NULL 
ORDER BY created_at DESC;
```

### Проверка адресов пользователя
```sql
-- В Supabase SQL Editor
SELECT * FROM addresses 
WHERE user_id = 'YOUR_USER_ID'
ORDER BY is_default DESC, created_at DESC;
```

### Логи сервера
Все важные события логируются в консоль:
- Запросы на восстановление пароля
- Ссылки для сброса пароля
- Ошибки API
